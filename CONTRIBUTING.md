# Contributing to Steve Jobs Thinking Advisor

Thank you for your interest in contributing to the Steve Jobs Thinking Advisor project! This document provides guidelines for contributing.

## How to Contribute

### 1. Reporting Issues
If you find bugs, errors, or have suggestions:
- Check if the issue already exists in the Issues section
- Create a new issue with clear description
- Include specific examples and context

### 2. Adding New Content
We welcome:
- **New cases**: Additional success or failure cases related to Steve Jobs
- **Tool improvements**: Enhancements to existing thinking tools
- **Knowledge graph expansion**: New entities, relationships, or attributes
- **Documentation improvements**: Better explanations, examples, or translations

### 3. Code Contributions
For code changes:
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## Content Guidelines

### Case Contributions
- Must be based on verifiable facts
- Include clear analysis of what worked/didn't work
- Connect to Jobs's thinking frameworks
- Provide actionable insights

### Tool Contributions
- Should be practical and actionable
- Include clear usage instructions
- Provide examples of application
- Connect to Jobs's actual practices

### Knowledge Graph Contributions
- Entities must be significant to Jobs's story
- Relationships should be well-documented
- Attributes should be factual and relevant
- Maintain consistency with existing data

## Quality Standards

### Accuracy
- All facts must be verifiable
- Quotes should be accurately attributed
- Historical events should be correctly dated
- Analysis should be balanced and objective

### Clarity
- Write in clear, concise language
- Use examples to illustrate points
- Structure content logically
- Make complex ideas accessible

### Completeness
- Provide sufficient context
- Include both positive and negative aspects
- Consider different perspectives
- Connect to broader themes

## Style Guidelines

### Writing Style
- Use active voice
- Be direct and concise
- Avoid jargon when possible
- Maintain consistent tone

### Formatting
- Use Markdown formatting
- Include headers for sections
- Use lists for multiple points
- Add links to sources when appropriate

### Bilingual Considerations
- Maintain parallel structure between English and Chinese versions
- Ensure accurate translation of key concepts
- Preserve the meaning and tone
- Update both versions when making changes

## Review Process

1. **Initial review**: Check for completeness and adherence to guidelines
2. **Fact-checking**: Verify accuracy of information
3. **Quality assessment**: Evaluate clarity and usefulness
4. **Integration**: Merge into main project with proper attribution

## Attribution

Contributors will be acknowledged in:
- The CONTRIBUTORS.md file
- Commit history
- Release notes

## Questions?

If you have questions about contributing, please open an issue or contact the maintainers.

Thank you for helping make this project better!
